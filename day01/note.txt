컴포넌트 : 웹을 이루는 부속, 컴퓨터 그래픽스에서 여러 개의 형상을 하나의 객체로 묶어 저장한 그룹. 주로 책상이나 의자 따위 같은 자주 사용되는 객체를 이 그룹으로 사용한다.

1. 리액트 (react.js)

- 자바스크립트 라이브러리 : 자바스크립트로 작성된 기능을 사용하도록 해주는 것
기능을 모아둔 파일
- 리액트는 현 시점, 가장 인지도가 높은 프론트엔드 라이브러리이다!
- 대표적인 Front-end 라이브러리 : 앵귤러, 뷰, 리액트 삼대장 중 리액트가 단연 넘버원!
- 페이스북에서 제작했다.

2. 리액트의 장점

- 렌더링 속도가 빠르다.
- 페이스북의 전폭적인 업데이트 지원.
- 컴포넌트 기반의 구조... 매우 직관적이다!
- 컴포넌트의 재사용이 쉽다!
- 인지도가 높다. == 자료가 방대하다, 취업문이 넓어진다, 수강생이 많아진다
- 모바일 앱을 개발할 때도 사용할 수 있어 스펙트럼이 넓다.

3. 리액트의 단점

- 리액트를 하기 위해 배워야 할 것이 아주 많다.
- html css javaScript node.js에서 npm이 어떻게 작동하는지 다 알아야한다
- 상태관리 복잡도가 높다. (요소의 동적인 변화를 주기가 까다롭다)
- 화살표 함수, 스프레드 연산, 전개 구문... 등등
 자바스크립트의 심화 문법이 익숙해야한다!

몇가지 시스템 명령어와 npm 프로젝트의 원리에 익숙해야한다
4. 리액트의 특징

- 요소를 생성하여 동적으로 렌더링하는 방식을 따른다. 이때 상태 관리가 렌더링의 기준이 된다.
- 정적 문서에 라이브러리 연계하여 작업하기는 복잡도가 너무 높다.
- 동적 렌더링을 위해서는 사용해야 할 추가 모듈이 더 많다.
- 다행히도, 리액트의 컴포넌트 작업을 위한 세팅이 모두 완료된 패키지가 존재한다.
- 패키지 이름은 : create-react-app
- 리액트는 SPA(Single Page Application)를 만든다.

5. npx create-react-app sample : sample이라는 이름의 표본 만들기  
  dir로 package.json 의 유무확인
  package.json이 있는 곳에 npm run build

- 리액트 관련 코드를 짜는 데 필요한 구조가 생성된다!
- 리액트 어플리케이션에 필요한 세부 모듈들이 설치가 되어진다!
- 기본적으로 완성된 샘플 앱이 제공된다!
